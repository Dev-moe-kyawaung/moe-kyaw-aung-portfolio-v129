## Version Guidelines

### Semantic Versioning
- **MAJOR** (1.0.0): Breaking changes, significant redesign
- **MINOR** (1.2.0): New features, backwards compatible
- **PATCH** (1.2.8): Bug fixes, minor improvements

### Release Types
- **Stable**: Tested, production-ready
- **Beta**: New features, may have bugs
- **Alpha**: Experimental, testing phase
- **RC**: Release candidate, nearly ready
